	- Removed temporary Safari note text below version row.

Version: 0.68.3e
Date: 2026-04-10

- [Add] Browser Detect debug utility (src/utils/browserDetect.js):
	- New detectBrowser() function returning { name, version, platform, userAgent }.
	- Detects Chrome, Safari, Firefox, Edge (including mobile variants CriOS, FxiOS, Edg).
	- Detects platforms: iOS, Android, Mac, Windows via navigator.userAgent.
- [Add] Browser debug display on login page (Login.vue):
	- Controlled by DEBUG_BROWSER = true flag (set false to hide).
	- Shows a small monospace line under the version number:
	  🔍 Browser: Safari 17 (iOS)
	- Minimal subtle styling; does not break layout.
	- Marked TEMP DEBUG in code comments for easy removal.
- [Change] Login diagnostics display refactor (Login.vue):
	- Removed giant inline diagnostics/error text block from the login layout.
	- Added compact warning state plus "View Login Diagnostics" button.
	- Added scrollable diagnostics modal with full details and close action.
	- Added in-modal "Copy Diagnostics" button.
	- Login page remains centered and usable even with long diagnostic content.
- [Fix] /login stylesheet MIME console error:
	- In themeDirectionSetting.js, LTR mode now removes rtlStyle href instead of setting href="".
	- Prevents browser from treating current route (/login) as a stylesheet URL.
	- Removed placeholder href values from dynamic stylesheet links in frontend/index.html.
- [Fix] Premature 401 protected API calls on login page:
	- App.vue now guards theme/profile settings calls so they run only on protected pages.
	- Protected shell components changed from v-show to v-if, preventing hidden mount-time API calls before auth.
	- Stops pre-login 401 spam for endpoints like /api/user-profile-settings and /api/notifications/unread-count.
- [Fix] Cross-site session persistence diagnostics and messaging:
	- Login flow now separates credential failures from post-login cookie/session persistence failures.
	- If login succeeds but follow-up /api/session has no cookie, UI now shows:
	  "Login succeeded, but the session cookie was not available for follow-up requests."
	- Added explicit diagnostics fields:
	  loginSucceeded, sessionCookiePersisted, sessionVerificationPassed.
	- Removed misleading fallback text that looked like bad credentials when session persistence failed.
- [Backend] Session/cookie flow hardening and debug telemetry:
	- Added proxy-aware session middleware behavior for secure cookies behind Render proxy.
	- Preserved production cookie policy for cross-origin requests (secure + sameSite none).
	- Added DEBUG-gated backend logs for login/session origin, session id, cookie header presence, and Set-Cookie diagnostics.
	- Updated /api/session diagnostic note for Safari/cloud browser cookie isolation scenarios.
- [Add] Temporary login-page backend connectivity debug tool:
	- Added compact debug panel with:
	  - "Test Backend API" button (calls /api/debug/ping)
	  - "Check Session API" button (calls /api/session with credentials)
	- Output now includes request URL, HTTP status, success/failure, response body, and network/timeout/CORS-style errors.
	- Output box is scrollable and compact; does not take over login layout.
- [Add] Temporary backend endpoint:
	- Added unauthenticated GET /api/debug/ping returning safe diagnostics:
	  ok, message, service, timestamp, env, origin.

Version: 0.68.4
Date: 2026-04-10

- [Fix] Login / Demo sign-in 404 errors resolved:
	- Demo login was calling /api/tmp-login (does not exist). Changed to /api/login.
	- Created frontend/.env.production with VITE_API_BASE=https://dev-asterisks-github.onrender.com
	  so the correct backend URL is baked into every production build.
	  Previously VITE_API_BASE was empty at build time, making all API calls hit the frontend server.
	- Added console.log output before all API calls showing API_BASE and full URL for devtools diagnosis.
- [Fix] Vite build exit code normalized:
	- Raised build.chunkSizeWarningLimit to 1500 kB so pre-existing large vendor chunks
	  do not cause a non-zero exit code during the Render deployment build step.
- [Change] Global UI scaling normalization:
	- Added html { font-size: 16px; -webkit-text-size-adjust: 100% } to style.css.
	- Added body { font-size: 1rem } baseline to prevent rem drift across browsers.
	- Prevents Safari mobile from auto-inflating text on narrow viewports.
- [Change] Login card width constrained:
	- Added max-width: 480px to .login-body so the card no longer fills the
	  full Bootstrap container on desktop screens (was stretching up to 1140px+).
